# 喪主マイページ 仕様（アット葬儀 bereaved.at-sougi.com 実地クロール結果）

調査日: 2026-07-18 / 実アカウント（雨宮 新一郎 様案件）を Playwright でクロール。
生キャプチャ: `docs/research/bereaved/`（個人情報を含むため gitignore 済み・コミット禁止）

## 0. 認証

| 項目 | 実装 |
|---|---|
| URL | `/account/sign-in?email=<ID>@at-sougi.com` |
| ID | 10桁ランダム英数字（例 `tp2sjzg2jp`）→ `<ID>@at-sougi.com` としてメール欄に投入 |
| 初期パスワード | **登録電話番号の下6桁**（例 320387） |
| ログイン後 | `/my/ceremonies/{ceremonyId}` へ直行（案件1件に1アカウント＝一覧を持たない） |
| パスワード変更 | `/my/update-password?ceremonyId={id}` |

ポイント: 喪主は「アカウント」ではなく「1葬儀」に紐づく。ceremonyId は UUID。

## 1. マイページ（ホーム）`/my/ceremonies/{id}`

見出し「〇〇 様向けページ」。カード型メニュー12件（全て `<button>`、`<a>` ではない）。
各カードは **タイトル＋バッジ（件数/状態）＋説明文** の3要素。

| カード | バッジ実例 | 遷移先 |
|---|---|---|

| 芳名録 | `27件` | `/attendees` |
| 入場記録 | `341件` | `/visitors` |
| 葬儀の写真 | `15枚` | `/funeral-photos` |
| アルバム | `26枚` | `/memorial-photos` |
| アカウント情報 | — | `/my/update-password?ceremonyId=` |
| メール通知設定 | — | `/mailAlarm-setting` |
| お問い合わせ | — | 外部 `at-sougi.com/contact` |
| 利用規約 | — | `/term` |
| 特商法表記 | — | `/transactions` |
| ログアウト | — | — |

## 2. 参列者へのご案内 `/announcement`

2つのURLを**コピーボタン付き**で提示するだけの画面。

- 訃報ページ `https://app.at-sougi.com/obituary/{id}`
- オンライン式場ページ `https://app.at-sougi.com/ceremony/{id}`

文言: 「参列者様に訃報ページ・葬儀式場をお知らせする際は、「URLをコピーする」ボタンを押してコピーしていただくか、各ページを開いて共有ボタンをご利用ください。」

## 3. オンライン式場詳細 `/online`

- **喪主挨拶**（全文表示）＋「喪主挨拶を変更する」ボタン
- **オンライン式場プレビュー**：「プレビューを生成する」ボタン（遅延生成）
- **オンライン式場URL**（リンク）
- **管理番号**：`不要`
- **公開期間**：`2026年07月14日(火) 12時50分 ~ 2026年09月12日(土) 12時50分 (60日間)` ← **開始/終了＋日数** を明示

喪主挨拶の既定文（そのまま初期値に使える）:
> 本日はお忙しい中、オンライン葬儀に参列頂きありがとうございました。
> 皆様にお見送りいただき、故人もさぞ喜んでいることと思います。
> ここに生前中賜りましたご厚誼に、厚く御礼申し上げます。
> 遺された私ども遺族に対しましても、変わらぬご指導ご厚誼を賜りますようお願い申し上げます。
> 本日は誠にありがとうございました。

## 4. 芳名録 `/attendees`

### 4-1. 上部：お香典精算テーブル（返礼品なし）

行 = `香典金額合計` / `香典決済代行手数料 (5%)` / `振込金額` / `振込予定日`
列 = **対象年月を3ヶ月分**（7月分・8月分・9月分）
振込予定日 = 対象月の**翌月末日**（07月分→2026年08月31日(月)）。曜日を括弧付きで表示。
注記2本：手数料5%の但し書き／振込日が祝休日なら翌営業日。

### 4-2. ダウンロード

- `CSVダウンロード` … **Shift_JIS**。列:
  `日時,姓,名,姓(ふりがな),名(ふりがな),故人とのご関係,会社名,郵便番号,住所,メールアドレス,電話番号,香典金額,決済ステータス,香典返し`
- `TXTダウンロード` … 1件1ブロックの人間可読形式（下記）

```
日時: 2026年07月14日(火) 16時42分
お名前: 中村 大（なかむら だい）
メールアドレス: dai@hokenshop.co.jp
電話番号: 09082603409
住所: 9140053 福井県敦賀市舞崎町2番地の8
故人とのご関係: その他
会社名: ほけんショップ株式会社
お香典: 0円
決済ステータス: 処理なし
メッセージ:
（本文）
```

### 4-3. 一覧

新しい順。1件 = `氏名 様` / 日時 / 会社名 / 関係（画像がある場合 `友人, 画像:4枚`）。
**20件ずつ「さらに表示（20件目まで表示中）」** で追加読み込み。下部に「入場記録へ」導線。

### 4-4. 参列者詳細 `/attendees/{epochSeconds}`

ID は**10桁エポック秒**（記帳時刻）。項目：名前／名前（ふりがな）／住所（〒＋住所）／故人とのご関係／会社名／メッセージ／画像。
未入力項目は**行ごと非表示**（会社名なしの人は会社名行が出ない）。

## 5. 入場記録 `/visitors`

「入場記録をダウンロード」＋ 一覧（`お名前不明` / 日時）。341件。記帳前の閲覧者も記録＝**アクセスログ**。

## 6. 葬儀の写真 `/funeral-photos` ／ アルバム `/memorial-photos`

同一UI。「最大30枚まで登録可能です。」＋ `追加`(input[type=file]) / `並び替え` / `削除` / `保存`。
- 葬儀の写真 = 式当日の写真
- アルバム = 故人の思い出写真

## 7. メール通知設定 `/mailAlarm-setting`

- 現在登録されているメールアドレス（未登録時「登録されているメールアドレスはありません。」）
- 「メールアドレスを登録/変更」
- 通知チェックボックス2種：
  - `receipt` … ご記帳の通知
  - `incense` … 香典決済の通知
