# アット葬儀 画面・フロー詳細仕様（実スクショ分析 2026-06-29）

出典: ユーザー提供スクショ `tmp/画面/*.png`（実運用テスト案件）。これが**クローンの最も正確な一次情報**。
公開ゲストアプリのドメイン: **`https://app.at-sougi.com/obituary/{uuid}`**（管理は `funeral.at-sougi.com/my/{uid}`）。
運営: Microwave Co.（フッター表記）。

## A. 全体構成（2系統）
- **管理画面（葬儀社/operator）**: 紫テーマ。`funeral.at-sougi.com/my/{uid}/...`
- **公開ゲスト画面（参列者）**: 紺×金×生成りテーマ・明朝見出し。`app.at-sougi.com/obituary/{uuid}`
- テスト案件は全画面上部に赤帯「この画面はテスト画面です」。

## B. 新規作成ウィザード（葬儀社が代理作成）
作成メニュー（「新しい葬儀を作成する」ドロップダウン）:
1. 訃報を作成する
2. 訃報＋オンライン式場を作成する
3. 訃報のテストを作成する（無料）
4. 訃報＋式場のテストを作成する（無料）

ステップ: **① 喪主／故人 → ② 訃報・香典 → ③ 記帳 → ④ 供花・供物 → ⑤ オンライン式場 →（葬儀サブ情報登録）**

### ① 喪主／故人 編集
- **写真で情報入力（試験運用中）**: 画像からAI-OCRで項目自動入力。
- 喪主: 姓/名/セイ/メイ（全必須）。
- **喪主管理画面ログインIDの発行（必須）**: ラジオ ⓐ自動IDで発行（電話番号ハイフンなし）/ ⓑ喪主メールアドレスで発行。
  ※オンライン香典を受ける場合、ここの電話/メール宛に口座情報の問い合わせが行く＝喪主＝受取人の連絡先。
- 故人: 姓/名/セイ/メイ（必須）, 没年月日, 没時間, 享年, 喪主との続柄。

### ② 訃報・香典 編集
- 訃報案内: 訃報タイトル(既定「訃報」), **テンプレート選択**, 訃報文(500字以内・例文プリセット), 「訃報を案内される喪主名」(必須), 喪式形式トグル(仏式/神式), 備考。
- **式1〜式5**（最大5式・折りたたみ）。各式:
  - 式名(必須)＝ボタン選択: 通夜式/告別式/葬儀告別式/一日葬/お別れの会/火葬式。
  - **日程調整中**チェック（未確定可）, 式日/式時刻(from-to)。
  - **場所調整中**チェック, 「**場所はひとつ前の式情報と同じ**」チェック。
  - 場所(会館・ホール), **登録式場を選択**, 郵便番号＋住所検索, 住所, 電話, FAX, URL, 備考。
- **香典決済オプション(必須)**: 香典決済 不要/必要・返礼品 不要/必要。
  注記:「喪主の口座設定完了後、**初回香典受付から30日後に受付終了**。それまで受付分は参列後に返金…」
- 御用達（葬儀社情報）: 担当者名/葬儀社名/電話/メール/URL。
- **訃報紙PDFプレビュー**（紙の訃報出力）。
- **【アプリ設定】訃報の共有文**: 共有URL `app.at-sougi.com/obituary/{uuid}` ＋共有本文テンプレ。

### ③ 記帳 / ④ 供花・供物 / ⑤ オンライン式場
- ③記帳: オンライン香典帳の記帳設定。
- ④供花・供物: 受付商品・締切設定。
- ⑤ オンライン式場（下記C-2）。
- 完了後 **葬儀詳細画面** へ遷移。喪主アカウントはここで初めて発行。

## C. オンライン祭壇（核心機能）

### C-1. 祭壇設定画面（レイヤー合成ビルダー）★最重要
遺影写真＋以下を選び**重ね合成**して3D祭壇画像を生成する:
| レイヤー | 選択肢 |
|---|---|
| 遺影(必須) | 仮画像を設定／写真を選択／編集（トリミング） |
| 額縁(必須) | 黒 / 黒(リボン) / 白 / 白(花) / グレー / 紫 / ピンク |
| 花飾り(左右)(必須) | 黒 / 白 / 花(1) / 花(2) |
| 祭壇(中央)(必須) | 焼香(黒) / 焼香(白) / 線香(1本) / 線香(2本) / 線香(3本) / 花(1) / 花(2) / 非表示 |
| 天板(必須) | 黒 / 木目 |
| 背景(必須) | 七宝 / 菊 / 波 / ドレープ(ベージュ) / ドレープ(ピンク) |
→ 当社実装: PNG透過レイヤーの重ね合わせ（CSS absolute or canvas）。素材セットをStorageに用意。

### C-2. オンライン式場設定（公開設定）
- オンライン式名(必須)（既定「故〇〇儀 葬儀会場」）。
- オンライン式場挨拶文(必須): 見出し(喪主挨拶)＋テンプレート選択＋本文＋喪主名。
- **公開日時(必須)**: 「喪主の同意後、すぐに公開する」チェック＋開始〜終了, **公開期間 1〜60日**。
- **入場画面の表示・入力制御(必須)**: 管理番号 不要/必要・参列者名 不要/必要（入場時に入力させるか）。
- **供花供物/贈答品の表示(必須)**: 表示しない/表示する。

### C-3. 公開ゲスト画面の構成（紺×金×生成り・明朝）
**(1) オンライン訃報案内ページ** `/obituary/{uuid}`:
- 白菊背景ヘッダー＋金枠内に「訃報」＋訃報本文＋喪主名。
- 「記」セクション → 式カード（式名・日時(令和表記)・式場(住所/〒/電話/URL/GoogleMapを開く)）。
- 儀式形態（仏式 等）。
- 「オンライン式場のご案内」＋公開期間＋**[オンライン式場]ボタン(金)**。
- 「知人にお知らせ」: Facebook / X / LINE / メール / SMS。
- 御用達（葬儀社情報）。
- フッター: @葬儀 ロゴ(紺地)・サイトポリシー/プライバシーポリシー/運営会社・Copyright Microwave Co.
- 下部固定バー: 「供花:MM/DD HH:mm まで」＋[ご注文はこちら](金)。

**(2) オンライン式場TOP（入場前）**: 故人名＋「葬儀会場」＋公開期間＋[入場]ボタン(金)。

**(3) オンライン式場（入場後＝祭壇）**:
- 合成された**3D祭壇画像**（遺影＋左右花＋中央焼香台＋天板＋背景）。
- **[お焼香をする]ボタン(金)**（バーチャル焼香）。
- 「ご記帳・メッセージ受付」＋[受付はこちら]（記帳＝香典帳＋メッセージ＋写真）。
- 「葬儀の写真」（円形サムネ）。
- 「アルバム」（グリッド／タップで拡大ライトボックス）。
- 「喪主挨拶」（本文＋喪主名）。
- 公開期間 / 知人にお知らせ(SNS) / フッター。

## D. 供花・供物 注文フォーム（ゲスト）
- 商品（画像・名称・説明・数量・単価税込・削除）, 商品点数, 合計金額。
- 注文者情報: お名前(姓名＋かな), 法人団体名, 郵便番号(住所検索), 住所, 電話, メール(確認再入力), 
  **札名(立札・旧字体希望チェック)**, 請求書・領収書宛名, 支払い方法(オンラインカード決済=1回払いのみ, VISA/Master/AMEX/Diners/Discover), 備考。
- 特定商取引法に基づく表記・プライバシーポリシー同意 → 入力内容を確認 → カード番号入力 → 完了。
- 「注文送信後、葬儀社が電話確認をもって注文完了」運用（与信→後確定）。

## F. 当社の改善ポイント（実物比較で確定）
1. レイヤー祭壇を**プレビュー即時反映**＋スマホ最適化＋reduced-motion。
2. 香典フローを Stripe Connect で**喪主直接帰属**（実物の「30日後受付終了・口座問い合わせ」運用より透明）。
3. 公開範囲を**4段階＋noindex既定**（実物はURL知る人のみ＋テスト帯）。
4. 宗派出し分けを仏式/神式だけでなく**浄土真宗/キリスト教式/無宗教**まで。
5. 焼香の**aria-live・連打抑止・累計表示**、記帳の承認制モデレーション。
6. 香典帳CSV/Excel（実物にもエクセルDLあり）＋**半返し自動計算**を追加。
