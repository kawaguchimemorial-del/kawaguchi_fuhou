# アット葬儀 実仕様リファレンス（クローン一次情報）

> 出典: ユーザー提供の実画面スクショ `tmp/画面/*.png`（実運用テスト案件）＋実ログイン調査。
> 公開ゲスト: `app.at-sougi.com/obituary/{uuid}` ／ 管理: `funeral.at-sougi.com/my/{uid}`。運営: Microwave Co.
> 詳細補足: `docs/04-atsougi-screens-spec.md`、`docs/03-atsougi-real-analysis.md`。
> **重要**: クローンを作る/直す前に必ず本書で実挙動を確認し、推測でUIを作らない。

## ⚠️ 実装で間違えやすい点（既知の取り違え）
- **葬儀一覧の「編集」/喪主名クリック → 「葬儀詳細」1枚ページに飛ぶ**。
  5ステップウィザードの再入力ではない。詳細ページは**セクション分割＋各セクションに「登録内容を編集」ボタン**で、
  セクション単位で編集する（編集は該当セクションのフォームに入る）。
- 新規作成は5ステップウィザード。作成完了後も**この葬儀詳細ページ**に着地する。

## 1. 葬儀一覧 `/my/{uid}/ceremonies`
- ヘッダー紫。右上「＋ 新しい葬儀を作成する」。
- 検索カード: ステータス(select) / 喪主氏名(漢字orカナ) / ◉喪主 ○故人 / ◉全て表示 ○本番のみ ○テストのみ / 検索ボタン。
- 「20件目まで表示中（更新日時の降順で表示中）」＋下部「さらに読み込む」。
- 各行の列: **タイプバッジ**(訃報のみ / 訃報+式場、テストは赤"テスト"バッジ) / 喪主名 / 故人名 /
  式1(式名+日付) / 式2(式名+日付 or -) / オンライン会場公開期間(開始〜終了) /
  注文受付終了日時(供花:/供物:/贈答品: の3段) / **ステータス**(公開中/公開終了/喪主アカウント発行待ち 等・赤文字) /
  香典決済オプション(利用する/利用しない) / メモ / **編集**(リンク)。
- 行クリック・「編集」→ **葬儀詳細ページ**へ。

## 2. 葬儀詳細ページ `/my/{uid}/ceremony/{id}`（★最重要・要再現）
**1枚の長いページ。セクションがアコーディオン(折りたたみ)で並ぶ。各セクション右上に状態バッジ（登録済/未登録/0件等）と、セクション本体に「登録内容を編集」紫ボタン。**

- ヘッダー: テスト葬儀バッジ＋「故 ●● 儀 葬儀詳細」＋葬儀社名。右上に [テスト/本番 切替] [喪主アカウント発行]。
- 黄色注意帯: 「ページは公開されていません。電話番号で喪主アカウントを発行してください」等。
- 「▶ 電話番号で喪主アカウント発行」ボタン（押すと喪主アカウントが初めて発行される）。

### セクション一覧（順序・内容）
1. **喪主／故人**（登録済）
   - 喪主: お名前 / セイ / 電話番号 / (ログイン)ID（発行済み喪主アカウント情報）
   - 故人: お名前 / セイ / 没日 / 享年 など
2. **訃報・香典**（登録済）
   - 訃報プレビュー: 「プレビューを表示する」
   - **訃報URL**: `https://app.at-sougi.com/obituary/{uuid}`
   - **訃報QR**: 「QRコードダウンロード」
   - **印刷ダウンロード**: [PDFダウンロード] [Wordダウンロード]
   - 訃報タイトル / 訃報文
   - 訃報を案内される喪主名
   - 式1: 日時 / 会場 / 〒 / 住所 / TEL / URL（式2〜5は「未登録」）
   - 儀式形態（仏式 等）
   - 御用達: 担当者名 / 葬儀社名 / 電話番号 / メールアドレス / URL
   - アプリ設定: 訃報の共有文（共有URL＋本文）
   - 香典決済オプション / 返礼品（利用する・利用しない）
3. **記帳**（登録済）— 参列者の記帳フォームで何を任意入力させるかの設定
   - 性別入力 / メールアドレス入力 / 電話番号入力 / 会社名 / メッセージ / 写真（各：任意入力 など）
4. **供花・供物**（登録済）
   - 供花の支払い終了日時 / 供物の支払い終了日時 / 注文フォーム入力・宛先表示入力
   - **商品テーブル**: 表示(✓) / 商品名 / 金額(税込) / 商品説明 / 商品メモ（例: 棺中花 22,000円, 中陽花…）
   - お悔やみのおくりものの表示
   - お名前(花形)等の設定
5. **オンライン式場**（登録済）
   - オンライン式場プレビュー: 「プレビューを表示する」
   - **オンライン式場URL** / **QRコードダウンロード**
   - 管理番号 / オンライン式名 / 挨拶文見出し / 挨拶文 / 公開期間
6. **香典**（未登録）— バーチャル参拝/香典の有無
7. **動画**（◯件）
8. **故人の写真**（◯件）
9. **アルバム**（◯件）
10. **贈答品**（未登録）— 表示名 / 配送 / 注文受付 / 注文フォーム入力・宛先表示
11. **個別情報（返礼等）の表示/設定変更**
12. **YouTubeライブ配信**（◯件）— ライブ配信はYouTube Live
13. **オンライン式場閲覧数一覧** — [入場一覧] [芳名録]
14. **供花の注文一覧** — 「安行終了 合計 0件 0円」/ [CSVダウンロード] [EXCELダウンロード]
    列: 商品 / 商品名 / 数量 / 金額 / 支払い方法 / ステータス / 喪主名 / 注文者 / 注文日時 / 法人・団体名 / メールアドレス / 領収書宛名
15. **供物の注文一覧**（同上の構造）
16. **贈答品の注文一覧**（同上の構造）
17. **注文メール通知先追加**

> 各セクションは独立編集（「登録内容を編集」で該当フォームへ）。状態バッジで登録済/未登録を表示。

## 3. 新規作成ウィザード（5ステップ）
① 喪主／故人 → ② 訃報・香典 → ③ 記帳 → ④ 供花・供物 → ⑤ オンライン式場 →（葬儀サブ情報登録）。
詳細項目は `docs/04-atsougi-screens-spec.md` B章参照。完了後は**葬儀詳細ページ**へ着地。
- 没年月日=カレンダー、没時間=時刻、式日/時刻も補助。会館はマスタ選択or手入力。
- オンライン式名は故人名から「故 ●● 儀　オンライン葬儀会場」を自動。
- 訃報文/挨拶文はテンプレート＋{{変数}}展開（{{故人続柄}}{{故人名}}{{没年月日和暦}}{{享年}}{{喪主名}}）。
- 「訃報を案内される喪主名」「挨拶文右下の喪主名」は喪主名から自動。

## 4. 公開ゲスト（参列者）
- **訃報案内** `/obituary/{uuid}`: 白菊背景＋金枠の訃報文＋喪主名 →「記」式カード(日時=和暦/会場/GoogleMap) →
  儀式形態 → オンライン式場のご案内(公開期間+[オンライン式場]) → 知人にお知らせ(FB/X/LINE/メール/SMS) →
  御用達 → フッター(@葬儀→自社版は川口典礼) → 下部固定[供花:締切まで／ご注文はこちら]。テスト時は赤帯。
- **オンライン式場TOP**: 故人名＋葬儀会場＋公開期間＋[入場]。
- **オンライン式場(入場後)**: レイヤー合成の祭壇 → [お焼香をする] → ご記帳・メッセージ受付 →
  葬儀の写真(円形) → アルバム(グリッド/拡大) → 喪主挨拶 → 公開期間 → SNS → フッター。
- **祭壇=レイヤー合成**: 遺影＋額縁7種＋花飾り左右4種＋中央祭壇8種(焼香/線香/花/非表示)＋天板2種＋背景5種。
- 供花注文フォーム: 商品/数量/単価税込, 注文者(姓名+かな/法人/〒+住所検索/TEL/メール確認再入力/**札名**(旧字体チェック)/請求書宛名/支払い=カード1回払い/備考), 特商法・プライバシー同意 → 確認 → カード入力 → 完了。

## 5. 設定 `/my/{uid}/setting`
アカウント設定 / 葬儀社管理 / ユーザー管理 / 供花・供物の設定・商品登録 / 贈答品の設定・商品登録 /
メール設定(注文通知先・自動送信メール文言)。

## 6. 配色・トーン
- ゲスト: 紺#1b2a4a × 金茶#a8842f/#b8943f × 生成り#faf8f3、明朝見出し。テスト帯=赤#d8443c。
- 管理: 紫#9b2fae 系。
- 葬送配慮: EC語回避・儀礼語・過度な演出禁止。

## 運用メモ
- 香典: 喪主口座設定後、初回受付から30日で受付終了。
- 供花等EC: 注文5日経過で自動売上計上（与信→確定の猶予）。
- 香典/供花/贈答品の各一覧は CSV/EXCEL ダウンロード可。
